JPSurv

Join Point Model for Relative Survival

Joinpoint models characterize trends over calendar time. The joinpoint model consists of linear segments connected through join points. The number and location of join points are estimated from data, and represent the times at which trends changed.

The JPSurv software has been developed to analyze trends in survival with respect to year at diagnosis. Survival data includes two temporal dimensions that are important to account for: the calendar year at diagnosis and the time since diagnosis. The JPSurv fits a Joinpoint survival model to the hazard of cancer death by year at diagnosis and assumes a common baseline hazard by time since diagnosis. In other words, the probabilities of dying at different time interval, e.g., 1 to 2 years, 2 to 3 years, and 4 to 5 years since diagnosis are proportional and share the same join points. The software uses discrete-time survival data, i.e. survival data grouped by years since diagnosis in the life table format. The software accommodates both relative survival and cause-specific survival. (Need to check this).

The JPSurv tool is useful to estimate when and how much survival changed over time and to predict survival into the future for simulation studies and scenario analyses.

Reading the data [Top]

The JPSurv requires grouped survival data. The data requires at a minimum the following variables: calendar year at diagnosis, survival time interval, number at risk at beginning of interval, number of deaths, number of cases lost to follow-up, and interval expected survival in the case of relative survival. The data may also include other covariates of interest such as cancer site, sex, stage etc...

To input data from a SEER*Stat dictionary file:

  1. Create a cause-specific survival or relative survival matrix from SEER*Stat. Make sure the Standard Life in the Parameters Tab is checked (JPsurv does not read the Summary Table format). In order to reflect survival in years since diagnosis, 12 months per interval needs to be specified. After generating the survival matrix, use the Export as text file option in the Matrix main menu option in SEER*Stat.
  2. In JPsurv go to the Import SEER*Stat matrix select......

Specifying the run [Top]

JPsurv uses a xxx algorithm to find the year/s at which significant changes in the hazard of death occurred (joinpoint years) and the number of Joinpoints. The user needs to specify the maximum number of joinpoints allowed (3 or more Joinpoints may take more than 5 minutes to run).

  • Select calendar year (independent variable)
  • Variables (cohort vs by variables)
  • Maximum number of joinpoints

Outputs [Top]

JPsurv outputs the number and location of join points, diagnostics of the model fit for each number of joints tested and summary measures of the trends in the hazard and/or cumulative survival for each segment between join points. The trend measures are summarized in the Trend Measures Table below. For the exact formulas refer to the paper.
